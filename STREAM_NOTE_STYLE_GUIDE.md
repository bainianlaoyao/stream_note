# Stream Note 风格设计系统（Clear 拟物）

## 1. 定义

本系统只使用两种基础设计概念：

- 底层是桌面与纸（Desk / Paper）。
- 其余所有交互实体都是桌面上的亚克力小块（Acrylic Blocks）。

这是一条强约束，不是建议。

---

## 2. 分层语义

### 2.1 L0 环境层（桌面）

- 作用：建立空间氛围与整体调性。
- 典型对象：页面舞台背景、环境光、整体底色。
- 原则：稳定、低噪声、不可喧宾夺主。

### 2.2 L1 内容承载层（纸）

- 作用：提供可阅读、可编辑的主工作面。
- 典型对象：Stream 主区域底面、侧边栏背景。
- 原则：强调可读性和连续性，不做强漂浮表达。

### 2.3 L2 交互物件层（亚克力块）

- 作用：承载按钮、卡片、标签、任务项等可操作对象。
- 典型对象：`ui-surface-card`、`ui-task-card`、`ui-btn`、`ui-pill`、`ui-chip`。
- 原则：材质统一、反馈明确、对比适中。

### 2.4 L3 浮起状态层（抬起的亚克力块）

- 作用：表达临时优先级与焦点切换。
- 典型对象：modal、popover、激活态卡片或面板。
- 原则：短时出现、轻量抬升、可快速回落。

---

## 3. 组件归属规则

1. Stream 页面背景和侧边栏背景属于 Desk / Paper，不属于亚克力块。
2. 任何需要“被点击、被选择、被切换”的对象，默认归类为亚克力块。
3. 同一组件在不同状态只能改变“高度感”，不能改变语义归属。
4. 新组件先回答“它是底面还是物件”，再进入视觉实现。

---

## 4. 视觉原则

### 4.1 材质原则

- 底层材质目标是安定和留白。
- 亚克力材质目标是识别和反馈。
- 禁止将底面容器视觉化为厚重悬浮卡片。

### 4.2 信息密度原则

- 内容优先，装饰退后。
- 强调状态使用最小足量策略，避免全局高对比。
- 同层级对象保持统一密度，不制造随机“重块”。

### 4.3 一致性原则

- 同语义对象共享同一套圆角、边界、反馈节奏。
- 跨页面保持同一视觉语法，避免每个页面独立设计语言。
- 历史 `glass-*` 命名只作为兼容，不再作为新设计术语。

---

## 5. 交互原则

1. 反馈节奏遵循“轻抬起 -> 稳定 -> 回落”。
2. Hover 表达可见但克制，不能抢占正文注意力。
3. Focus 与 Active 必须可区分，优先保证可访问性。
4. 动效只服务于状态确认，不用于炫技。

---

## 6. 可读性与无障碍原则

- 长文本区域始终优先阅读舒适度。
- 交互控件在任意背景下都需要可识别边界。
- 状态变化不能只依赖颜色，需要同时有形态或结构提示。

---

## 7. 设计评审检查表

1. 该对象是否正确归类为 Desk / Paper 或 Acrylic Blocks？
2. 是否存在把底层容器做成漂浮块的语义错误？
3. 同级对象的视觉重量是否一致？
4. 交互反馈是否清晰且不干扰阅读？
5. 是否复用现有语义实体，避免新增无意义视觉类型？

---

## 8. 文档维护约束

- 本文档只记录设计概念和原则，不包含 CSS、组件代码或框架实现细节。
- 实现细节由前端代码知识文档和代码仓内样式文件维护。
- 当语义模型调整时，先更新本文档，再同步具体实现文档。
